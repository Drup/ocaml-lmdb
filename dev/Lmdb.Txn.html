Module Lmdb.Txn

module Txn: sig .. end

A series of operations performed atomically.

type -[< `Read | `Write ] t

A transaction handle. A transaction may be read-only or read-write.

val go : ([< `Read | `Write ] as 'a) Lmdb.cap ->
       Lmdb.Env.t -> ?txn:'a t -> ('a t -> 'b) -> 'b option

go cap env ?txn f makes a transaction in env with the capabilities cap and using the function f txn.

The function f will receive the transaction handle txn. All changes to the environment env done using the transaction handle txn will be persisted to the environment when f returns. After f returned, the transaction handle is invalid and should therefore not be leaked outside f.

  • Returns None if the transaction was aborted with abort, and Some _ otherwise.
txn : Create a child transaction to txn. Here is an example incrementing a value atomically: 
go rw env begin fun t ->
let v = Db.get ~txn k in
Db.put ~txn k (v+1) ;
v
end
      
val abort : [< `Read | `Write ] t -> 'b

abort txn will abort the transaction.
